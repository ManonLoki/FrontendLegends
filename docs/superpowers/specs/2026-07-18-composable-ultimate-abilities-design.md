# 组合式绝招能力设计

## 目标

把现有连击、异常、削减上限和巨额伤害四种互斥绝招，整理为由数据声明、按对应门派架构功法等级成长的能力模型：

1. 连击：攻击 3～6 次，次数随对应内功等级提高。
2. 异常：从麻痹、虚弱、中毒中必定附加 1～2 种不同状态，数量随对应内功等级提高。
3. 必中增伤：攻击必定命中，伤害倍率随对应内功等级从 1.5 倍提高到 2.0 倍。
4. 吸取：命中后按实际伤害吸取体力，或按目标最大精力的比例吸取精力。

必中增伤与吸取能力还要成为普通攻击招式可以显式配置的附加能力。玩家与 NPC 使用同一套数据、成长和结算规则。

## 明确保留的现有行为

`scripts/combat/combat_rules.gd` 中按 20/40/50/70/80/90 级给普通攻击招式随机附加麻痹、虚弱或中毒的 `ATTACK_MOVE_STATUS_TABLE` 保持不变。普通攻击招式原有的状态触发概率、1～3 回合持续时间和伤害折算保持不变。

香草派绝招的“必定异常”是新的绝招能力，与普通攻击招式现有的随机异常并存，不取代或削弱后者。

## 方案选择

采用数据驱动的组合特性方案。绝招数据声明各档名称与能力，运行时根据门派架构功法当前等级计算派生数值。普通招式通过可选的 `combatEffects` 字段复用必中增伤和吸取能力。

不采用以下方案：

- 继续扩展 `kind` 分支：改动较少，但普通招式无法干净复用能力，组合效果会继续增加硬编码。
- 让每个绝招拥有全部效果：规则表面简单，但会抹平四门派差异并造成数值叠乘。

## 门派能力分配

| 门派 | 一档绝招 | 二档绝招 | 核心能力 |
|---|---|---|---|
| NG神教 | 依赖注入·定海 | AOT·万法归一 | 连击 |
| 香草派 | 闭包·封天 | 原生·归真一击 | 必定异常 |
| 鱿鱼山庄 | 响应·千丝 | Vite·瞬息万变 | 必中增伤 |
| 量子仙宗 | 钩式·千手 | 并发·量子千幻 | 一档吸血、二档吸精 |

一档仍在门派架构功法 30 级解锁，二档仍在 80 级解锁。80 级后两档同时保留，不以二档覆盖一档。

## 等级与成长

所有成长使用对应的、已装备门派架构功法等级 `inner_level`，范围钳制为 30～100。基础架构等级仍通过现有公式提供绝招攻击力加成，但不决定特性成长。

统一成长进度为：

```text
progress = clamp((inner_level - 30) / 70, 0, 1)
```

### 连击

| 内功等级 | 攻击次数 | 每击倍率 | 理论总倍率 |
|---|---:|---:|---:|
| 30～49 | 3 | 65% | 195% |
| 50～74 | 4 | 60% | 240% |
| 75～99 | 5 | 55% | 275% |
| 100 | 6 | 50% | 300% |

每击独立命中、闪避、招架、暴击、加力、装备和普通攻击招式结算；目标倒下后立即停止后续攻击。加力继续按每次实际命中分别支付精力。

### 必定异常

- 30～79 级：从麻痹、虚弱、中毒中随机选择 1 种。
- 80～100 级：随机选择 2 种不同异常。
- 所选异常不再进行第二次概率判定，均附加 2 回合。
- 异常能力本身必定生效；附带的伤害仍按普通命中流程结算。即使附带伤害被闪避，绝招声明的异常仍会施加。
- 重复状态继续使用现有规则：取原剩余回合数和新回合数的较大值，不叠加回合数。

### 必中增伤

伤害倍率线性成长：

```text
damage_scale = 1.5 + progress * 0.5
```

“必中”跳过基础命中失败和防守方闪避招式，但保留基础招架、招架招式减伤、暴击、装备、状态、伤势和加力阶段。这样必中属于主动招式的显式命中修正，不绕过其余战斗输入。

### 吸血与吸精

吸血比例：

```text
drain_hp_ratio = 0.20 + progress * 0.15
```

吸血量以目标本次实际损失的体力为基数，不使用超过目标剩余体力的溢出伤害。恢复后不得超过攻击者本场体力上限。

吸精比例：

```text
drain_mp_ratio = 0.08 + progress * 0.07
```

吸精基数为目标本场最大精力。实际转移量同时受目标当前精力和攻击者缺失精力限制。目标没有精力、精力已空或攻击者精力已满时，转移量为零。吸取只转移已有资源，不凭空生成精力。

吸血和吸精均要求附带攻击实际命中；被闪避时不产生吸取。

## 数据模型

`assets/Data/skills.json` 的绝招定义改为按档声明能力，运行时字典携带以下稳定字段：

```text
id, name, tier, inner_level, inner_power, mp_cost, abilities
```

`abilities` 是能力标识数组，当前支持：

```text
multi, abnormal, guaranteed_hit, drain_hp, drain_mp
```

派生次数、倍率、状态数量和吸取比例由窄规则模块根据 `inner_level` 计算，不重复写入 JSON。

普通招式可以增加可选字段：

```json
{
  "combatEffects": {
    "guaranteedHit": true,
    "damageScale": 1.5,
    "drainHpRatio": 0.2,
    "drainMpMaxRatio": 0.08
  }
}
```

普通招式不支持 `multi` 或绝招式的必定异常字段。第一轮实现只建立并验证这些可选字段的运行能力，不擅自给现有普通招式批量分配新效果；现有普通招式行为保持不变。

## 代码边界

- 新增独立、低于 300 行的绝招/招式能力规则模块，负责成长计算、能力查询和资源转移值计算。
- `scripts/combat/ultimate_actions.gd` 负责绝招流程编排，不再保存各门派硬编码倍率。
- `scripts/combat_system.gd` 继续保持命中、招架、暴击、伤害及资源扣除的权威阶段，只增加显式攻击选项和普通招式 `combatEffects` 读取。
- `scripts/skills/skill_loadout.gd` 和 `scripts/combat/enemy_ai.gd` 对称构建玩家与 NPC 绝招，均携带门派架构等级。
- `scripts/battle/battle_hud_renderer.gd` 从绝招标准字典格式化真实能力说明，消除当前展示值与执行值不一致。
- 不修改受保护的 `scripts/game_battle_ui.gd`、`scripts/ui_progress_meter.gd` 或 `scripts/game.gd` HUD 布局代码。

## 战斗顺序与不变量

普通攻击核心阶段继续保持：命中、基础招架、暴击、减伤与浮动、招式/加力/装备/状态修正、扣除资源与体力。

新增能力在对应显式阶段结算：

1. 必中作为主动招式命中选项进入命中阶段。
2. 伤害倍率进入招式修正阶段。
3. 先扣除目标实际体力，再以实际体力损失计算吸血。
4. 吸精在攻击命中并完成伤害后转移双方现有精力。
5. 必定异常在香草派绝招动作完成时统一附加。

四属性、已装备功法、装备、状态、伤势、精力、加力及 NPC 显式配置仍参与战斗。玩家和 NPC 使用相同的能力规则，不改变 `combatRank` 职责，不改变存档字段或资源身份。

## 错误与边界处理

- 未装备门派架构功法、等级未达门槛、绝招不存在或精力不足时，沿用现有拒绝行为。
- 未知能力标识被忽略并产生可测试的空效果，不崩溃。
- 全部比例钳制为非负安全范围；次数限制为 3～6，状态数量限制为 1～2。
- 切磋模式继续保证目标最低 1 点体力；吸血只按切磋中实际扣除的体力计算。
- 精力转移不得令任一方精力小于 0 或超过最大值。

## 测试与验收

测试驱动实现需要覆盖：

1. 30/50/75/100 级连击分别为 3/4/5/6 次，并使用对应每击倍率。
2. 30/79 级必定得到 1 种异常，80/100 级必定得到 2 种不同异常。
3. 香草派异常在附带伤害闪避时仍生效。
4. 必中攻击无法被基础命中或闪避招式避开，但仍可被招架减伤。
5. 必中伤害在 30/100 级分别为 1.5/2.0 倍。
6. 吸血基于实际体力损失，并受攻击者体力上限约束。
7. 吸精基于目标最大精力，并受双方当前/最大精力约束。
8. NPC 与玩家使用同一能力成长结果。
9. 普通招式可通过 `combatEffects` 使用必中增伤、吸血和吸精。
10. `ATTACK_MOVE_STATUS_TABLE` 映射及普通招式既有随机异常回归保持通过。
11. 绝招菜单展示与运行时真实数值一致。
12. 无存档结构变化，旧存档中的技能等级与装备状态仍可直接构建新绝招。
