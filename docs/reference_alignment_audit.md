## 核验结果

### 保留的内容与语义

- 人物、技能、物品、任务和地图继续使用原有稳定 ID；显示名、门派、身份和主要对白保持原世界观语气。
- 技能主题、装备槽、师徒教学、冥想资格、任务类型、地图交互与冷却语义继续沿用既有结构。
- 全局时钟及连续动作维持本项目调度：学习/冥想 30 Hz，练功 5 Hz；学习 tick 不重复推进全局时钟。
- TMX 对象坐标、对象 ID 与 `Props/props` 大小写等 Godot 适配继续保留；地图运行时仍直接读取 TMX/TSX。
- 新角色初始 Token 为 50；技能状态使用蛇形英文键；装备按当前设计只存在于本局内存。

### v4 有意改变的数值

- 基础攻击、防御和体力已改为低基数、软缩放公式；精力对体力只按平方根贡献，不再线性制造数千生命池。
- 命中底线提高，并把攻击、闪避和招架招式拆成三条触发曲线，减少连续空回合。
- 玩家与 NPC 的已装备功法战斗字段统一进入聚合；加力和绝招的高数值使用递减收益或平方根。
- 66 名 NPC 获得唯一四维配点、`combatRank` 与 `combatRole`；原总点数保留，但不再共用同一模板。
- 小学生等非战斗单位不再成为高耐久目标，也不进入随机击杀池；未满 18 岁人物、商贩、典当商、师父、全部任务发布/交付端点、当前任务目标和保护人物同样被过滤。
- 战斗恢复受次数和最大体力比例限制，战斗 HUD 激活时生存时钟暂停。
- 普通击杀奖励改按武学评级与阶位计算，和 NPC 随身财富分离；`combatReward` 仍可显式覆盖。
- 环任务和悬赏成长由复利改为带上限的线性增长；物品原价报销与真正奖励分开，普通悬赏击杀和交付奖励互斥。
- 战败资源损失降为各 5%～10%，且最多令一门功法下降一级；战后伤势按净体力损失结算并封顶。致命战斗的濒死/归零破相分别按 15%/30% 独立判定，每次永久损失 20 容貌。

### 数据同步结论

- `docs/data/FrontendLegendsData.xlsx` 是人物、物品、技能、任务和世界事件的维护入口，可往返生成五份运行时 JSON。
- `docs/balance_design.md` 记录 v4 公式、目标区间、经济约束、模拟结果和研究依据。
- 任何未来“对齐参照项目”的修改都必须先判断是否会破坏 v4 的体验目标，不能以逐字节一致为理由覆盖当前运行时。
