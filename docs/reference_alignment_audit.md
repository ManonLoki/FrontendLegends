# 参照项目设定一致性检查

参照项目：`/Users/manonloki/Documents/study/game/FrontendLegends`

原则：人物/NPC、文案、数据和计算公式以参照项目为准；全局时钟以及冥想、学习、练功的推进频率和调度方式以本 Godot 项目为准。

## 检查清单

- [x] 核心数据表：NPC、技能、任务、道具字段、价格、掉落、售卖和教学配置
- [x] 人物设定：四维、初始资源、基础属性成长、称号、生存容量、体力/精力上限
- [x] NPC 设定：四维、技能、装备、金钱、门派、身份、拜师条件、地图实例覆盖
- [x] 交谈与提示文案：NPC 台词、任务台词、地图告示、战报、交易和物品反馈
- [x] 战斗公式：攻击、防御、命中、闪避、暴击、招架、先手、逃跑、伤害波动
- [x] 战斗扩展：被动招式、异常状态、绝招、加力、疗伤、战败惩罚、战利品
- [x] 技能公式：评级、学习需求、学费、潜能、派生属性、拜师/学习上限
- [x] 生存与物品：食水消耗、回血、伤势、年龄、冷却、性别限制、买卖价格
- [x] 任务公式：固定奖励、环任务、悬赏缩放、冷却、目标生成与交付条件
- [x] 地图交互：NPC/告示属性、传送目标和交互文案；允许 Godot 为碰撞对齐调整坐标
- [x] 例外项：冥想/学习/练功及其全局时钟推进保留 Godot 优化实现

## 核验结果

- `npcs.json`、`skills.json`、`quests.json` 与参照项目逐字节一致。
- `items.json` 的有效数据一致；唯一差异是 `_note` 改写为 Godot 文件/脚本路径说明。
- 地图对象的交互文本和属性内容一致。部分对象坐标、对象 ID 和 `Props/props` 大小写为 Godot 碰撞与运行时适配，不属于设定差异。
- 发现并修复一处重构回归：NPC 被动招式仍使用旧触发率和旧异常档位。现已恢复为参照项目的 `25% + 每招 4%`、封顶 `55%`，异常档位为 `20/40/50/70/80/90`。
- 全局时钟及连续动作维持本项目频率：学习/冥想 30 Hz，练功 5 Hz；学习 tick 不重复推进全局时钟。
- 经逐项确认，玩家精力上限已由原 Godot 的“修为×2”改为与参照项目一致的“修为×1”；NPC 精力改用装备内功综合等级、25 点单位和架构修正计算。
- 新角色初始 Token 已对齐为 50；读档会裁剪姓名、钳制资源/食水、清理未知物品与冷却，并迁移旧档缺失的入门师父。
- 玩家与 NPC 武学称号已改用参照项目的完整称号表，只统计已装备武学并排除灵感。
- TMX 对 NPC 展示字段的覆盖机制按用户决定暂不迁移，保留为已知运行时能力差异。
- 背包购买、出售、丢弃和卸装反馈文案已对齐参照项目；按用户决定继续保留本项目的 `stackLimit` 限制。
- 小不二悬赏奖励已对齐为经验基数 ×2、潜能 ×3、Token ×2.4；暗网榜单复用接取文案。
- 活阎王生死簿目标池已对齐为不排除发布者本人；任务放弃功能按用户决定不迁移。
- 赛博传送费用已对齐为当前人物精力上限的三分之一（向上取整）。
- 基础编码与基础招架对编码属性的每十级反哺已改为累加，不再互相覆盖。
- 完整武学称号造成的长战报已限制在战斗面板报告区域内并自动换行。
- 同门改拜的师父高低已改为按教学表显式 `maxTeachLevel` 比较；技能书会执行道具自己的 `maxLearnLevel`。
- 功法装备会拒绝异门功法，且装备成功/失败反馈标点已对齐参照项目。
- 冥想资格现在同时要求装备基础架构与本门高级架构；练功推进/升级/到顶文案已对齐。
- 玩家及 NPC 的装备四维加成会进入战斗快照和战斗体力上限，但不改写永久四维。
- 背包与典当条目改为按道具 ID 稳定排序；按用户决定，已装备物品仍须先卸下才能丢弃。
- 动态悬赏 NPC 会读取运行时 `defaultLine`；每次重新进入 Game 场景会清空 NPC 击杀隐藏记录。
- 存档只持久化基础四维并在读档时重算派生四维；小不二奖励对齐为经验/潜能 ×3、Token ×2.4，九日环浮动奖励使用向下取整。
- 技能书完整学习门槛尚未设计完成，按用户决定暂不补齐四维、综合等级、平均等级和前置技能校验。
