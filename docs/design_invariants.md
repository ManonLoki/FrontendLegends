# 不可变设计规则

本文档定义 FrontendLegends 的基础玩法契约。数值、系数、阈值、封顶值和计算公式可以调优，但不得借调参改变这里规定的资源身份、输入维度、阶段顺序或系统边界。

## 修改授权

若修改会改变本文件中的任一不变量，首次提出修改不视为授权。执行者必须先停止写入，列出：被触及的不变量、准确文件、行为变化、存档/数据影响和拟更新的回归测试，并请求用户单独确认。只有用户在后续消息中明确确认后才能修改；确认仅对列出的改动生效一次。

以下调整无需解锁：在保留全部输入、资源身份、阶段顺序和显式修正来源的前提下，调整数值常量、曲线指数、倍率、上下限、概率、时间间隔或计算公式。任何“公式调整”若实际删除输入、交换职责、合并资源、绕过阶段或引入隐藏倍率，仍属于不变量修改。

## 学习系统不变量

师父学习必须保持以下流程，顺序不可交换：

```text
潜能 → 按灵感转换为学习经验 → 固定学习经验满
     → 按本级实际潜能消耗结算 Token → 技能升级
```

1. 潜能、学习经验、Token 是三种不同数据，不能互相改名、共用字段或直接等同。
2. 每一级固定学习经验只由功法数据和目标等级决定；同一功法同一级在任何灵感下必须相同，数值必须为偶数，并严格高于前一级。
3. 灵感只影响潜能转换学习经验的效率。灵感越高，完成同一级所需的学习时间片和潜能必须越少。
4. Token 学费依据本级实际消耗的潜能结算；在相同功法和等级下，高灵感的 Token 消耗必须更少。
5. 学习经验未满不能升级；经验满但 Token 不足时保留经验和实际潜能消耗记录，不得免费升级。
6. 秘籍升级是独立路径，可以不消耗潜能和 Token，但必须清理该技能未完成的师父学习记录。

允许调整：固定经验曲线、每级增幅、灵感转换公式、转换倍率上下限和 Token 比例。禁止未经确认改成“灵感直接降低固定升级经验”、直接以潜能填等级、经验未满升级、固定 Token 与实际潜能脱钩，或改变下节固化的持续动作频率。

## 持续动作与冥想不变量

1. 学习固定为 10 Hz，冥想固定为 10 Hz，练功固定为 5 Hz；三个动作分别使用独立累加器和固定时间片。
2. 学习 tick 不得重复推进全局时钟；有效冥想 tick 推进 `1/10` 秒，有效练功 tick 推进 `1/5` 秒。
3. 冥想必须保持“恢复当前精力 → 精力满 → 当前精力清零 → 精力修为 +1”的顺序。修为提升的 tick 必须以当前精力 0 结束，下一 tick 才能从 0 继续恢复。
4. 长帧补算若在本帧触发修为提升，必须先停止本帧冥想循环并渲染归零状态；未消费的累积时间保留到下一帧，不得跨过 0 直接显示为正数。

未经确认不得改变上述频率、交换冥想阶段、在修为提升时保留旧精力，或在同一帧越过归零展示边界。

## 战斗系统不变量

战斗结果必须由显式输入共同产生：

```text
四属性
+ 已装备功法的等级、战斗加成与已解锁招式
+ 装备提供的属性和战斗加成
+ 当前状态、伤势、精力、加力与主动招式
+ NPC 显式 combatRank、combatRole、AI 和 combatReward 约束
→ 命中、招架、暴击、减伤、伤害、资源变化与胜负
```

1. 四属性都必须保留战斗作用：编码影响攻击与招架；思维影响命中/回避关系、先手和逃跑；架构影响防御、体力及精力修为链；灵感影响暴击。可以调整公式，不能让任一属性退出战斗。
2. 功法只有装备后才进入常规战斗聚合；已装备功法的攻击、防御、命中、闪避、招架、精力上限、伤势减免和招式必须按其显式数据生效。
3. 玩家和 NPC 必须使用同口径的四属性、功法和装备聚合。允许数据不同，不允许一方绕过某类输入或使用未声明的隐藏倍率。
4. 装备的四属性与 `attack/defense/hit/dodge/crit/parry/woundInflict` 等显式字段必须进入对应结算，不能只显示在面板而不生效。
5. 状态、伤势、当前精力、加力和主动招式属于战斗修正层，必须在明确阶段结算。普通攻击顺序保持“命中 → 基础招架 → 暴击 → 减伤与浮动 → 招式/加力/装备/状态修正 → 扣除资源与体力”。
6. `combatRank` 只缩放 NPC 体力与普通奖励；`combatRole` 和 AI 是显式原型/行为配置，不得暗中修改攻击、防御、命中或伤害。
7. 战斗公式可以调参，但角色差异必须能追溯到四属性、已装备功法、装备、状态、阶位、AI 或其他明确数据字段。

允许调整：各属性系数、曲线、概率、伤害倍率、阶位倍率、招式数值、状态强度与回合数。未经确认不得删除属性作用、让未装备功法生效、取消装备/状态层、改变阶位职责、加入不可追溯倍率或改变核心结算阶段。
